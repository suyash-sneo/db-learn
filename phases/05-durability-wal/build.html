Build: a checksummed WAL with transactional page allocation

Goal: Make page updates and allocation atomic across clean restart and the documented crash model: append committed page images to a WAL, merge reads, discard invalid tails, and checkpoint only with the required synchronization order.
⚠️ AI off

This is safety-critical database code. The page defines formats, contracts, failing tests, and todo!() stubs only. Implement WAL parsing, checksums, commit, recovery, and checkpoint yourself.

Layering decision

WalPager<D, W> owns the main-database and WAL BlockIo backends directly. It reuses Phase 2's 32-byte database header and page numbering, but replaces the raw pager's write path so no uncommitted page can be evicted into the main file. Higher layers must use Txn for page allocation, metadata updates, reads, and writes.

MiniDB WAL format

The shape follows SQLite's 32-byte WAL header and 24-byte frame header, but uses MiniDB's own magic/version. All stored integers and checksum words are big-endian.

WAL header offsetBytesField
040x4d444257 (“MDBW”)
44version 1
84page size
124checkpoint sequence
164salt 1
204salt 2
248checksum pair over bytes 0..24
Frame offsetBytesField
04page number
44database page count after commit; 0 for non-commit frame
88two salts copied from WAL header
168cumulative checksum pair
24page sizecomplete page image

The checksum uses SQLite's two-accumulator recurrence over pairs of big-endian u32 words. A frame checksum covers its first 8 header bytes plus page data, seeded by the previous checksum; salts are validated separately. [src: SQLite database file format §4.2 — WAL checksum algorithm] This is a distinct project format with one fixed byte order.

Transaction contract

  • PageNo = u32. Byte offsets use checked u64.
  • Txn::allocate_page increments a transaction-local page count, stages a zero page, and stages page 1 with the updated Phase 2 page_count. It does not extend the main database before commit/checkpoint.
  • Reads see transaction-dirty pages, then committed WAL frames, then the main database.
  • An empty commit is a no-op. A nonempty commit marks only its last frame with the post-commit page count.
  • Durability::Full synchronizes the WAL before success. Durability::Relaxed omits that per-commit sync and promises only process/clean-restart persistence until a later synchronization/checkpoint.
  • Recovery validates the header and frames, retains state through the last valid commit, and truncates an invalid/uncommitted tail before accepting new writes. While a valid committed WAL exists, its final commit marker is authoritative for logical page count even if an interrupted checkpoint left the main-file header or physical length behind.

Checkpoint contract

  1. Synchronize the WAL, including commits made in relaxed mode.
  2. Extend the main database to the committed logical length, then copy the newest committed frame for each page. The WAL remains authoritative if a crash interrupts either operation.
  3. Synchronize the main database.
  4. Only then reset the WAL and clear/rebuild the in-memory index. This project truncates the WAL to zero; a later open creates and synchronizes a fresh header before a Full commit can succeed.

Acceptance criteria

  • All supplied tests pass unchanged.
  • A transaction can allocate and commit pages without touching out-of-range main-db pages first.
  • Rollback, uncommitted tails, checksum failures, and truncated frames are invisible after reopen.
  • A nonempty full-size WAL header with bad magic/version/page-size/checksum is rejected rather than treated as an empty log.
  • A prior committed transaction survives when a later commit frame is torn.
  • A complete checkpoint leaves the database usable with an empty WAL.
  • A simulated interruption after copying only some checkpoint pages remains recoverable because the WAL was not reset.
  • Manual external check: use real files and a failpoint-driven child process; kill it at commit and checkpoint boundaries, reopen repeatedly, and verify only pre-commit or fully committed states. MemBlock tests validate logic, not real fsync or power-loss behavior.
  • Manual lifecycle check: document the platform-specific directory synchronization required when creating/removing a separate WAL file.

#[cfg(test)]
mod tests {
    use super::*;
    use pager::{MemBlock, Pager};

    fn fresh() -> WalPager<MemBlock, MemBlock> {
        let p = Pager::create(MemBlock::empty(), 4096, 4).unwrap();
        WalPager::open(p.into_inner(), MemBlock::empty(), Durability::Full, [11, 22]).unwrap()
    }
    fn reopen(w: WalPager<MemBlock, MemBlock>) -> WalPager<MemBlock, MemBlock> {
        let (db, wal) = w.into_blocks();
        WalPager::open(db, wal, Durability::Full, [33, 44]).unwrap()
    }

    #[test]
    fn allocated_committed_page_survives_reopen() {
        let mut w = fresh();
        let mut tx = w.begin();
        let p = tx.allocate_page().unwrap();
        tx.write_page(p, vec![0xab; 4096]).unwrap();
        tx.commit().unwrap();
        let mut w = reopen(w);
        assert_eq!(w.read_committed(p).unwrap(), Some(vec![0xab; 4096]));
        assert_eq!(w.page_count(), 2);
    }

    #[test]
    fn rollback_and_uncommitted_tail_are_invisible() {
        let mut w = fresh();
        { let mut tx = w.begin(); let p = tx.allocate_page().unwrap(); tx.write_page(p, vec![1; 4096]).unwrap(); tx.rollback(); }
        assert_eq!(w.page_count(), 1);
        w.append_uncommitted_for_test(2, &vec![2; 4096]).unwrap();
        let mut w = reopen(w);
        assert_eq!(w.read_committed(2).unwrap(), None);
    }

    #[test]
    fn torn_later_commit_preserves_prior_commit() {
        let mut w = fresh();
        let p2 = { let mut tx = w.begin(); let p = tx.allocate_page().unwrap(); tx.write_page(p, vec![3; 4096]).unwrap(); tx.commit().unwrap(); p };
        let p3 = { let mut tx = w.begin(); let p = tx.allocate_page().unwrap(); tx.write_page(p, vec![4; 4096]).unwrap(); tx.commit().unwrap(); p };
        let (db, mut wal) = w.into_blocks();
        wal.data.truncate(wal.data.len() - 17);
        let mut w = WalPager::open(db, wal, Durability::Full, [1, 2]).unwrap();
        assert_eq!(w.read_committed(p2).unwrap(), Some(vec![3; 4096]));
        assert_eq!(w.read_committed(p3).unwrap(), None);
    }

    #[test]
    fn checksum_failure_discards_later_transaction() {
        let mut w = fresh();
        let p2 = { let mut tx = w.begin(); let p = tx.allocate_page().unwrap(); tx.write_page(p, vec![8; 4096]).unwrap(); tx.commit().unwrap(); p };
        let p3 = { let mut tx = w.begin(); let p = tx.allocate_page().unwrap(); tx.write_page(p, vec![9; 4096]).unwrap(); tx.commit().unwrap(); p };
        let (db, mut wal) = w.into_blocks();
        let last = wal.data.len() - 1;
        wal.data[last] ^= 0x80;
        let mut w = WalPager::open(db, wal, Durability::Full, [1, 2]).unwrap();
        assert_eq!(w.read_committed(p2).unwrap(), Some(vec![8; 4096]));
        assert_eq!(w.read_committed(p3).unwrap(), None);
        assert_eq!(w.page_count(), 2);
    }

    #[test]
    fn bad_nonempty_wal_headers_are_rejected() {
        for offset in [0, 4, 8, 24] {
            let w = fresh();
            let (db, mut wal) = w.into_blocks();
            wal.data[offset] ^= 0xff;
            let opened = WalPager::open(db, wal, Durability::Full, [1, 2]);
            assert!(opened.is_err(), "corruption at WAL header offset {offset} was accepted");
        }
    }

    #[test]
    fn checkpoint_leaves_db_self_sufficient() {
        let mut w = fresh();
        let mut tx = w.begin(); let p = tx.allocate_page().unwrap(); tx.write_page(p, vec![5; 4096]).unwrap(); tx.commit().unwrap();
        w.checkpoint().unwrap();
        let (db, wal) = w.into_blocks();
        assert_eq!(wal.data.len(), 0);
        let mut w = WalPager::open(db, wal, Durability::Full, [7, 8]).unwrap();
        assert_eq!(w.read_committed(p).unwrap(), Some(vec![5; 4096]));
    }

    #[test]
    fn interrupted_checkpoint_keeps_wal_recoverable() {
        let mut w = fresh();
        let mut tx = w.begin();
        let a = tx.allocate_page().unwrap(); let b = tx.allocate_page().unwrap();
        tx.write_page(a, vec![6; 4096]).unwrap(); tx.write_page(b, vec![7; 4096]).unwrap(); tx.commit().unwrap();

        let mut w = reopen(w);
        assert_eq!(w.read_committed(a).unwrap(), Some(vec![6; 4096]));
        assert_eq!(w.read_committed(b).unwrap(), Some(vec![7; 4096]));
    }

Stuck? — graduated hints

Recovery keeps an uncommitted transaction
Nudge: A valid frame is not necessarily committed.
Bigger: First scan and remember the byte boundary of each valid commit.
Technique: rebuild the final page index only through the last commit boundary, then truncate the WAL there before future appends.
Allocation writes the main file too early
Nudge: Page count is transactional metadata.
Bigger: Keep a logical count in Txn and stage page 1 like any other dirty page.
Technique: the main file is extended only during checkpoint; committed reads beyond its current physical end are served from WAL frames.
Torn-tail test loses the earlier transaction
Nudge: The test must tear a later transaction.
Technique: commit transaction A, commit B, truncate B's final frame, then require A to survive and B to disappear.
Checkpoint interruption loses pages
Nudge: Do not modify/reset the safety net early.
Technique: sync WAL → copy → sync db → reset. The test helper stops after copy, before db sync/reset, so reopen must still use the unchanged WAL.
