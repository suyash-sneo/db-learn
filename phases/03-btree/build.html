Build: persistent B+tree search, insert, and range scan

Goal: Build an ordered byte-string map whose nodes live in Phase 2 pager pages. Implement an exact node codec, point lookup, duplicate-key replacement, byte-aware leaf and internal splits, root growth, persistence, and inclusive range scan.
⚠️ AI off

The B+tree is a prohibited solution component. This page defines the format and behavior, then supplies only todo!() stubs, failing tests, and graduated hints. You write every encoding, search, and split operation.

Scope

This build requires get, insert, and inclusive range. Inserting an existing key replaces its value and returns the previous value. Full B+tree deletion, redistribution, merge, and root shrink are not part of this milestone; lesson 3.6 provides the later design vocabulary. Overflow values arrive in Phase 4.

Required Phase 2 contract

Use the reviewed pager API: PageNo = u32, page 1 reserved for metadata, read_page returns owned bytes, modify_page edits through a closure, allocate_page returns a zeroed data page, and the header exposes root_page/set_root_page. Do not extend page 1 again.

Exact node format

All integers are big-endian. Every node occupies one complete pager page. Bytes not used by the encoded node are zero. The 16-byte header is:

OffsetBytesMeaning
01tag: 1 leaf, 2 internal
11reserved, must be zero
22entry/separator count
42end of slot array (free_start)
62start of cell-content area (cell_start)
84leaf: next leaf or 0; internal: rightmost child
124reserved, must be zero
  • Immediately after the header are sorted 2-byte cell offsets, one per entry/separator.
  • Leaf cell: key_len:u16 | value_len:u16 | key | value.
  • Internal cell: left_child:u32 | key_len:u16 | separator_key. The header stores the final/rightmost child.
  • Leaf keys and internal separators are strictly increasing.
  • Internal invariant: children.len() == separators.len() + 1.
  • Separator separators[i] equals the smallest key reachable from children[i + 1]. Therefore c₀: K<k₀, cᵢ: kᵢ₋₁≤K<kᵢ, and cₙ: K≥kₙ₋₁.

Size and split rules

  • For a leaf entry, needed = 2-byte slot + 4-byte lengths + key bytes + value bytes. Reject it when needed > (page_size − 16) / 4. This project limit is intentionally conservative so one entry cannot consume most of a page.
  • A node fits only when its complete encoded representation—including header, slots, lengths, child pointers, keys, and values—fits in one page.
  • When a variable-size node overflows, choose a split boundary near half of the encoded bytes, not merely half of the entry count. Both halves must be nonempty and encodable.
  • Leaf split: right leaf receives the upper entries; repair next links; copy the right leaf's first key into the parent.
  • Internal split: promote one separator; left/right nodes retain the children and separators on either side. Check the invariant after constructing both nodes.

Ordered milestones

  1. M1 — Owned node codec. Encode/decode LeafNode and InternalNode; reject malformed offsets, lengths, tags, ordering, child counts, null children, and pages whose fields overlap.
  2. M2 — Create/open. Create an empty leaf root and persist its page number in Phase 2 metadata. Open validates that the stored root is an allocated data page and decodes it.
  3. M3 — Search. At internal nodes choose partition_point(|sep| sep <= key); binary-search leaves.
  4. M4 — Insert/replace without split. Preserve strict key order. Existing key replacement returns the previous value without creating a duplicate.
  5. M5 — Leaf split. Select a byte-aware boundary, create the right page, repair leaf links, and propagate the copied separator.
  6. M6 — Internal split/root growth. Propagate upward using an owned path stack; when the old root splits, allocate and publish a new root.
  7. M7 — Range/persistence. Descend once to lo, follow next-leaf links through hi, then sync and reopen through the pager metadata.

Acceptance criteria

  • All supplied tests pass unchanged.
  • Malformed node pages return Corrupt; decoding never panics on arbitrary bytes.
  • Random-order insertion forces leaf/internal/root splits and every inserted value is found.
  • Duplicate insertion replaces exactly one value and preserves one key.
  • Inclusive range output is strictly sorted, contains exactly the requested keys, and follows leaf links after the initial descent.
  • Sync plus reopen preserves root metadata and data.
  • Manual integration check: repeat persistence with FileBlock and a unique temporary path. Phase 3 still claims only clean-restart persistence, not crash atomicity.

append as an inline unit-test module in src/btree.rs 
#[cfg(test)]
mod tests {
    use super::*;
    use pager::MemBlock;

    fn fresh() -> BTree<MemBlock> {
        let pager = Pager::create(MemBlock::empty(), 4096, 8).unwrap();
        BTree::create(pager).unwrap()
    }

    #[test]
    fn node_codec_roundtrips_both_types() {
        let leaf = Node::Leaf(LeafNode {
            next: 9,
            entries: vec![
                (b"a".to_vec(), b"one".to_vec()),
                (b"z".to_vec(), b"two".to_vec()),
            ],
        });
        let internal = Node::Internal(InternalNode {
            children: vec![2, 3, 4],
            separators: vec![b"m".to_vec(), b"t".to_vec()],
        });
        for node in [leaf, internal] {
            let page = encode_node(&node, 4096).unwrap();
            assert_eq!(decode_node(&page, 20).unwrap(), node);
        }
    }

    #[test]
    fn decoder_rejects_arbitrary_non_node_page() {
        assert!(matches!(
            decode_node(&vec![0u8; 4096], 10),
            Err(BTreeError::Corrupt(_))
        ));
    }

    #[test]
    fn random_order_insert_get_forces_split_cascades() {
        let mut t = fresh();
        let mut keys = (0u32..5000).collect::<Vec<_>>();
        keys.sort_by_key(|x| x.wrapping_mul(2654435761));
        for i in &keys {
            let k = i.to_be_bytes();
            assert_eq!(t.insert(&k, &k).unwrap(), None);
        }
        for i in 0u32..5000 {
            let k = i.to_be_bytes();
            assert_eq!(t.get(&k).unwrap().as_deref(), Some(&k[..]));
        }
        assert_ne!(t.root_page(), 2); // the original leaf root split
    }

    #[test]
    fn duplicate_key_replaces_without_duplicate() {
        let mut t = fresh();
        assert_eq!(t.insert(b"k", b"old").unwrap(), None);
        assert_eq!(t.insert(b"k", b"new").unwrap(), Some(b"old".to_vec()));
        assert_eq!(t.range(b"k", b"k").unwrap(), vec![(b"k".to_vec(), b"new".to_vec())]);
    }

    #[test]
    fn range_is_inclusive_sorted_and_exact() {
        let mut t = fresh();
        for i in 0u32..1000 {
            let k = i.to_be_bytes();
            t.insert(&k, &k).unwrap();
        }
        let out = t.range(&100u32.to_be_bytes(), &200u32.to_be_bytes()).unwrap();
        assert_eq!(out.len(), 101);
        assert!(out.windows(2).all(|w| w[0].0 < w[1].0));
        assert_eq!(out.first().unwrap().0.as_slice(), &100u32.to_be_bytes());
        assert_eq!(out.last().unwrap().0.as_slice(), &200u32.to_be_bytes());
        assert!(t.range(b"z", b"a").unwrap().is_empty());
    }

    #[test]
    fn rejects_oversized_entry() {
        let mut t = fresh();
        let big = vec![0u8; 2048];
        assert!(matches!(
            t.insert(b"k", &big),
            Err(BTreeError::EntryTooLarge { .. })
        ));
    }

    #[test]
    fn sync_and_reopen_preserve_root_and_values() {
        let mut t = fresh();
        for i in 0u32..2000 {
            let k = i.to_be_bytes();
            t.insert(&k, &k).unwrap();
        }
        let root = t.root_page();
        t.sync().unwrap();
        let io = t.into_pager().into_inner();
        let pager = Pager::open(io, 8).unwrap();
        let mut reopened = BTree::open(pager).unwrap();
        assert_eq!(reopened.root_page(), root);
        assert_eq!(reopened.get(&1777u32.to_be_bytes()).unwrap(), Some(1777u32.to_be_bytes().to_vec()));
    }
}

Stuck? — graduated hints

M1: decoder safety
Nudge: Treat every on-disk offset and length as hostile.
Bigger: Validate header size, tag, reserved bytes, slot-array end, cell-area start, every slot offset, and every checked cell end before slicing.
Technique: decode into owned vectors only after all bounds for that field are proven; finish by checking strict key order and internal child invariants.
M3: equal separator keys go to the wrong child
Nudge: The right child owns keys equal to its minimum separator.
Bigger: Count separators ≤ key, not separators < key.
Technique: partition_point(|sep| sep.as_slice() <= key).
M5: choosing a split by count still overflows
Nudge: Two entries can have very different encoded sizes.
Bigger: Evaluate candidate boundaries using the same complete encoded_len calculation used by the codec.
Technique: choose a nonempty boundary whose left/right byte totals are close and whose two encodings both fit; do not duplicate encoding arithmetic in split code.
M6: parent updates fight the borrow checker
Nudge: Do not hold page borrows while moving upward.
Bigger: Store owned path entries such as (parent_page_no, chosen_child_index).
Technique: decode one page into an owned Node, mutate/encode/write it, drop that value, then load the next parent.
