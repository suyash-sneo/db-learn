Build: a JSON document collection with CRUD, an index, and a filter planner

Goal: Layer one collection over the Phase 4 record store: validate and encode documents, assign integer ids, rebuild derived indexes on open, maintain a sparse secondary index, evaluate a precisely defined filter language, and prove index plans return the same rows as a scan.
⚠️ AI off

The evaluator, index maintenance, and planner are database-component logic. This page supplies formats, contracts, todo!() stubs, failing tests, and graduated hints—not a working implementation.

Scope and honest layering

This build uses UTF-8 JSON through serde_json::Value. The top level must be an object. That keeps the bytes inspectable, but it does not provide BSON types or a native binary value. A future codec change requires a persisted format version and migration.

The Phase 3 and Phase 5 implementations are not yet wired into one reusable stack, so this exercise keeps primary and secondary indexes in memory and rebuilds them from record bytes on open. It does not claim crash atomicity: a failed heap mutation can leave duplicate/orphan records until the real Phase 5 transaction layer is integrated. Successful operations must keep in-memory state coherent; open must reject duplicate ids as corruption.

Fixed project semantics

  • Ids: _id is an i64. Auto ids begin at 1 and increase with checked arithmetic. An explicit id advances the next auto id if necessary. Deleted ids are not reused. Exhaustion returns an error.
  • Replace: replace(id, doc) inserts id when doc omits _id, and rejects a different supplied id.
  • Index definitions: the caller passes indexed field paths to create/open. Definitions are not persisted in this exercise; a real catalog is a required later integration. Open rebuilds exactly the requested indexes.
  • Sparse secondary index: a missing path has no entry; explicit JSON null does. Each key maps to a sorted set of ids, preventing duplicates.
  • Indexable values: null, bool, integer, and string. Floats, arrays, and objects receive no secondary-index entry.
  • Field paths: split on .; reject empty segments. Object segments are exact keys; array segments are non-negative decimal usize indexes. A failed step is missing. Keys containing a literal dot are not addressable in this version.
  • Eq/Ne: deep JSON equality/inequality, but any comparison leaf with a missing field is false.
  • Ranges: only integer-to-integer and string-to-string. All type mismatches and unsupported values are false.
  • In: the query value must be an array; the field matches when deeply equal to one member. This build does not implement MongoDB's array-field expansion.
  • Planner: only an indexable Eq, same-type range, or In leaf may generate candidates. For top-level And, choose the first usable indexed leaf. All other filters scan. Every candidate is fetched and checked against the complete filter.

Ordered milestones

  1. M1 — Lifecycle. Compose Pager::create/open with RecStore::open; implement scan; rebuild the primary index, requested secondary indexes, and next-id state. Reject malformed JSON, non-object documents, bad ids, and duplicate ids during open.
  2. M2 — CRUD. Normalize ids, serialize before mutating storage, insert/get, replace by inserting new bytes then retiring the old record, and delete. Update every in-memory index on successful operations.
  3. M3 — Evaluator. Resolve dot paths through objects and numeric array positions; implement the exact missing/type/Eq/Ne/range/In semantics above plus And/Or/Not/All.
  4. M4 — Scan oracle. Evaluate every primary id in order. This is the correctness reference.
  5. M5 — Secondary index and planner. Build sparse indexes, report the selected Plan, generate/deduplicate candidate ids, then recheck the full predicate.

Acceptance criteria

  • All nine supplied tests pass unchanged.
  • Auto and explicit ids cannot collide or wrap; duplicate ids and invalid documents are rejected.
  • Replace preserves the requested id; delete and replace remove stale secondary entries.
  • Missing, explicit null, cross-type ranges, Ne, Not, and In follow the written contract.
  • Equality and range index plans return exactly the scan oracle's ids.
  • Reopen reconstructs primary/secondary indexes and the next auto id from live records.
  • Manual integration check: route record and persisted B+tree mutations through one Phase 5 transaction before claiming crash atomicity; persist collection/index definitions in a catalog.

#[cfg(test)]
mod tests {
    use super::*;
    use pager::MemBlock;
    use serde_json::json;

    fn fresh(indexes: &[&str]) -> DocStore<MemBlock> {
        DocStore::create(
            MemBlock::empty(),
            4096,
            8,
            indexes.iter().map(|s| (*s).to_string()).collect(),
        ).unwrap()
    }
    fn ids(rows: Vec<(i64, Value)>) -> Vec<i64> {
        rows.into_iter().map(|(id, _)| id).collect()
    }
    fn eq(field: &str, value: Value) -> Filter {
        Filter::Compare { field: field.into(), op: Cmp::Eq, value }
    }

    #[test]
    fn ids_are_assigned_advanced_and_not_duplicated() {
        let mut s = fresh(&[]);
        assert_eq!(s.insert(json!({"x": 1})).unwrap(), 1);
        assert_eq!(s.insert(json!({"_id": 10, "x": 2})).unwrap(), 10);
        assert_eq!(s.insert(json!({"x": 3})).unwrap(), 11);
        assert!(matches!(
            s.insert(json!({"_id": 10})),
            Err(DocError::Duplicate(10))
        ));
        assert_eq!(s.get(11).unwrap().unwrap()["_id"], json!(11));

        let mut exhausted = fresh(&[]);
        exhausted.insert(json!({"_id": i64::MAX})).unwrap();
        assert!(matches!(
            exhausted.insert(json!({"x": 1})),
            Err(DocError::IdExhausted)
        ));
    }

    #[test]
    fn invalid_documents_and_conflicting_replace_ids_fail() {
        let mut s = fresh(&[]);
        assert!(matches!(s.insert(json!([1, 2])), Err(DocError::BadDocument(_))));
        assert!(matches!(s.insert(json!({"_id": "x"})), Err(DocError::BadDocument(_))));
        let id = s.insert(json!({"x": 1})).unwrap();
        assert!(matches!(s.replace(id, json!({"_id": id + 1})), Err(DocError::BadDocument(_))));
        s.replace(id, json!({"x": 2})).unwrap();
        assert_eq!(s.get(id).unwrap().unwrap()["_id"], json!(id));
    }

    #[test]
    fn comparison_contract_covers_missing_types_not_and_in() {
        let doc = json!({
            "n": 5,
            "nil": null,
            "s": "five",
            "nested": { "items": [{ "v": 7 }] }
        });
        let cmp = |field: &str, op, value| Filter::Compare { field: field.into(), op, value };
        assert!(DocStore::<MemBlock>::matches(&eq("nested.items.0.v", json!(7)), &doc));
        assert!(!DocStore::<MemBlock>::matches(&cmp("missing", Cmp::Ne, json!(1)), &doc));
        assert!(DocStore::<MemBlock>::matches(&Filter::Not(Box::new(eq("missing", json!(1)))), &doc));
        assert!(!DocStore::<MemBlock>::matches(&cmp("s", Cmp::Gt, json!(4)), &doc));
        assert!(DocStore::<MemBlock>::matches(&cmp("n", Cmp::In, json!([4, 5, 6])), &doc));
        assert!(DocStore::<MemBlock>::matches(&eq("nil", Value::Null), &doc));
    }

    #[test]
    fn equality_index_matches_scan_and_omits_missing() {
        let mut s = fresh(&["city"]);
        s.insert(json!({"city": "London"})).unwrap();
        s.insert(json!({"city": null})).unwrap();
        s.insert(json!({"name": "missing city"})).unwrap();
        let f = eq("city", Value::Null);
        assert_eq!(s.plan(&f), Plan::Index { field: "city".into() });
        assert_eq!(ids(s.find(&f).unwrap()), ids(s.find_scan(&f).unwrap()));
        assert_eq!(ids(s.find(&f).unwrap()).len(), 1);
    }

    #[test]
    fn range_index_with_residual_matches_scan() {
        let mut s = fresh(&["age"]);
        for (age, city) in [(25, "London"), (31, "London"), (40, "Paris")] {
            s.insert(json!({"age": age, "city": city})).unwrap();
        }
        let f = Filter::And(vec![
            Filter::Compare { field: "age".into(), op: Cmp::Gt, value: json!(30) },
            eq("city", json!("London")),
        ]);
        assert_eq!(s.plan(&f), Plan::Index { field: "age".into() });
        assert_eq!(ids(s.find(&f).unwrap()), ids(s.find_scan(&f).unwrap()));
        assert_eq!(ids(s.find(&f).unwrap()).len(), 1);
    }

    #[test]
    fn or_uses_scan() {
        let s = fresh(&["city"]);
        let f = Filter::Or(vec![eq("city", json!("London")), eq("city", json!("Paris"))]);
        assert_eq!(s.plan(&f), Plan::FullScan);
    }

    #[test]
    fn replace_and_delete_remove_stale_index_entries() {
        let mut s = fresh(&["city"]);
        let a = s.insert(json!({"city": "London"})).unwrap();
        let b = s.insert(json!({"city": "London"})).unwrap();
        s.replace(a, json!({"city": "Paris"})).unwrap();
        s.delete(b).unwrap();
        let london = eq("city", json!("London"));
        assert_eq!(ids(s.find(&london).unwrap()), Vec::<i64>::new());
        assert!(s.get(b).unwrap().is_none());
    }

    #[test]
    fn reopen_rebuilds_indexes_and_next_id() {
        let mut s = fresh(&["city"]);
        s.insert(json!({"_id": 40, "city": "London"})).unwrap();
        let io = s.into_inner();
        let mut s = DocStore::open(io, 8, vec!["city".into()]).unwrap();
        assert_eq!(ids(s.find(&eq("city", json!("London"))).unwrap()), vec![40]);
        assert_eq!(s.insert(json!({"city": "Paris"})).unwrap(), 41);
    }

    #[test]
    fn reopen_rejects_duplicate_ids_as_corruption() {
        let mut s = fresh(&[]);
        s.rec.insert(br#"{"_id":5,"copy":1}"#).unwrap();
        s.rec.insert(br#"{"_id":5,"copy":2}"#).unwrap();
        let io = s.into_inner();
        assert!(matches!(
            DocStore::open(io, 8, vec![]),
            Err(DocError::Corrupt(_))
        ));
    }

Stuck? — graduated hints

Open rebuilds the wrong next id
Nudge: Derive it from validated live documents, not slot numbers.
Bigger nudge: Track the maximum explicit _id while scanning.
Technique: start at Some(1); for every id at or above the current candidate, use checked_add(1). i64::MAX changes the state to None, meaning auto-id exhaustion.
Indexed and scanned range results disagree
Nudge: Storage order is broader than range-predicate semantics.
Bigger nudge: A query age > 30 must only scan the Key::Int subrange, never later enum variants such as strings.
Technique: construct same-variant lower/upper Bounds, deduplicate ids, then run matches(full_filter, doc) anyway.
Replace leaves two copies after an I/O error
Nudge: No operation ordering makes two record-store writes crash-atomic.
Bigger nudge: The exercise intentionally exposes the composition gap.
Technique: keep successful in-memory publication ordered and make open reject duplicate ids; the actual fix is to execute insert-new, index changes, and retire-old through one Phase 5 transaction—not to invent a fragile cleanup sequence.
Missing fields accidentally match null
Nudge: Option::None and Some(Value::Null) are different.
Technique: field resolution returns None for a missing path; every Compare returns false immediately. An explicit null proceeds to Eq/Ne and receives a Key::Null secondary entry.
