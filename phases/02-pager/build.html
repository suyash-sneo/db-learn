Build: a single-file pager with a buffer pool

Goal: Turn one byte store into a checked, numbered array of fixed-size pages with a reserved metadata page, bounded caching, dirty tracking, deterministic eviction, and an explicit synchronization operation.
⚠️ AI off

Use the lessons, primary documentation, compiler errors, and the hint ladder. This page fixes the format and API contracts, then gives you only failing tests and todo!() stubs. You must write the pager implementation.

Scope and non-goals

This phase proves clean create/open/reopen behavior. It does not provide transactions, atomic commits, concurrent access, or power-loss recovery. Dirty eviction writes directly to the main file only because no WAL exists yet. Phase 5 must change that write path before uncommitted data is allowed to coexist with a recovery protocol.

Fixed file-format decisions

  • PageNo = u32. Page numbers are 1-based; 0 is the null pointer.
  • Page 1 is entirely reserved for database metadata. Ordinary data pages begin at page 2. Higher layers must never place a tree node or record in page 1.
  • The project accepts power-of-two page sizes from 512 through 32768 bytes. This is the curriculum's simpler u16 encoding, not SQLite's full page-size encoding.
  • All multi-byte header fields are big-endian and encoded field-by-field.
  • The 32-byte header occupies the start of page 1; the rest of page 1 is zero/reserved:
    OffsetBytesField
    016magic: "minidb format 1\0"
    162format version: 1
    182page size
    204page count, including page 1
    244root page, 0 when absent
    284freelist head, 0 when absent
  • A file length must equal page_count × page_size on clean open. A mismatch is corruption/truncation, not permission to guess.

Chosen ownership model

read_page returns an owned Vec<u8>. Mutations happen through modify_page(page_no, closure), and the closure cannot retain the borrowed slice. This deliberately avoids an escaping page guard in the first pager. Lesson 2.5 still teaches pin counts because a later zero-copy or concurrent cache would need them; this build does not pretend a Boolean pin flag is sufficient.

Ordered milestones

  1. M1 — Block I/O. Implement exact positional reads/writes, length query/change, and sync for MemBlock and FileBlock.
  2. M2 — Header codec. Encode/decode the exact 32-byte layout. Validate magic, version, page size, page count, pointer ranges, and whole-file length.
  3. M3 — Create/open. Creation writes and synchronizes page 1. Open never silently initializes an empty or malformed file.
  4. M4 — Checked page I/O. Reject page 0, page 1 through the public data-page API, out-of-range pages, wrong-sized buffers, and offset overflow.
  5. M5 — Cache. Load misses, return owned read copies, mutate through the closure, and mark edited frames dirty.
  6. M6 — Eviction. Use one documented deterministic policy (FIFO is enough). In this pre-WAL build, write a dirty victim before reusing its frame.
  7. M7 — Allocate/sync/reopen. Append a zeroed page, mark metadata dirty, flush dirty data plus page 1, call BlockIo::sync, then prove reopen works.

Acceptance criteria

  • The supplied unit tests pass without changing their assertions.
  • You add the same create/write/sync/reopen scenario for FileBlock using a unique temporary path, and remove the file afterward.
  • Page 1 cannot be read or modified through the ordinary data-page API.
  • Dirty data survives eviction and a subsequent clean reopen.
  • Opening bad magic, bad version, bad page size, impossible pointers, or a file-length mismatch returns a typed error.
  • Manual check: inspect FileBlock and confirm its sync method calls File::sync_all. This is an OS synchronization request, not a claim of transactional atomicity or a universal hardware guarantee.
  • Manual check: document that durable creation of a new directory entry may require a platform-specific containing-directory sync; that operation is outside this narrow BlockIo trait.

Failing tests

append as an inline unit-test module in src/lib.rs 
#[cfg(test)]
mod tests {
    use super::*;

    fn fresh(capacity: usize) -> Pager<MemBlock> {
        Pager::create(MemBlock::empty(), 4096, capacity).unwrap()
    }

    #[test]
    fn create_then_open_really_roundtrips() {
        let p = fresh(2);
        let io = p.into_inner();
        let reopened = Pager::open(io, 2).unwrap();
        assert_eq!(reopened.page_size(), 4096);
        assert_eq!(reopened.page_count(), 1);
        assert_eq!(reopened.root_page(), 0);
    }

    #[test]
    fn rejects_bad_magic_and_page_size() {
        let p = fresh(2);
        let mut bad_magic = p.into_inner();
        bad_magic.data[0] ^= 0xff;
        assert!(matches!(
            Pager::open(bad_magic, 2),
            Err(PagerError::BadMagic)
        ));

        let p = fresh(2);
        let mut bad_size = p.into_inner();
        bad_size.data[18..20].copy_from_slice(&1000u16.to_be_bytes());
        assert!(matches!(
            Pager::open(bad_size, 2),
            Err(PagerError::BadPageSize(1000))
        ));
    }

    #[test]
    fn rejects_bad_version_pointer_and_file_length() {
        let p = fresh(2);
        let mut bad_version = p.into_inner();
        bad_version.data[16..18].copy_from_slice(&2u16.to_be_bytes());
        assert!(matches!(
            Pager::open(bad_version, 2),
            Err(PagerError::UnsupportedVersion(2))
        ));

        let p = fresh(2);
        let mut bad_pointer = p.into_inner();
        bad_pointer.data[24..28].copy_from_slice(&2u32.to_be_bytes());
        assert!(matches!(
            Pager::open(bad_pointer, 2),
            Err(PagerError::BadPointer { field: "root_page", page: 2 })
        ));

        let p = fresh(2);
        let mut truncated = p.into_inner();
        truncated.data.pop();
        assert!(matches!(
            Pager::open(truncated, 2),
            Err(PagerError::FileLengthMismatch { .. })
        ));
    }

    #[test]
    fn rejects_null_reserved_and_out_of_range_pages() {
        let mut p = fresh(2);
        assert!(matches!(p.read_page(0), Err(PagerError::NullPage)));
        assert!(matches!(p.read_page(1), Err(PagerError::ReservedPage(1))));
        assert!(matches!(p.read_page(2), Err(PagerError::PageOutOfRange(2))));
    }

    #[test]
    fn first_allocated_data_page_is_two() {
        let mut p = fresh(2);
        assert_eq!(p.allocate_page().unwrap(), 2);
        assert_eq!(p.page_count(), 2);
        assert_eq!(p.read_page(2).unwrap(), vec![0; 4096]);
    }

    #[test]
    fn dirty_page_survives_eviction() {
        let mut p = fresh(1);
        let a = p.allocate_page().unwrap();
        let b = p.allocate_page().unwrap();
        p.modify_page(a, |bytes| bytes[0] = 42).unwrap();
        let _ = p.read_page(b).unwrap(); // capacity 1: evicts a
        assert_eq!(p.read_page(a).unwrap()[0], 42);
        p.sync().unwrap();
        let mut reopened = Pager::open(p.into_inner(), 1).unwrap();
        assert_eq!(reopened.read_page(a).unwrap()[0], 42);
    }

    #[test]
    fn sync_then_reopen_preserves_data_and_metadata() {
        let mut p = fresh(2);
        let root = p.allocate_page().unwrap();
        p.modify_page(root, |bytes| bytes[7] = 99).unwrap();
        p.set_root_page(root).unwrap();
        p.sync().unwrap();

        let io = p.into_inner();
        let mut reopened = Pager::open(io, 2).unwrap();
        assert_eq!(reopened.root_page(), root);
        assert_eq!(reopened.read_page(root).unwrap()[7], 99);
    }

    #[test]
    fn offset_math_is_one_based_and_checked() {
        let p = fresh(2);
        assert_eq!(p.offset(1).unwrap(), 0);
        assert_eq!(p.offset(4).unwrap(), 3 * 4096);
        assert!(matches!(p.offset(0), Err(PagerError::NullPage)));
    }
}

Stuck? — graduated hints

M1: exact I/O and extension
Nudge: Your trait promises either the complete requested transfer or an error.
Bigger: For a cursor-based file backend, seek before read_exact/write_all. set_len changes the logical length; allocation still writes a full zero page so every backend has the same observable contents.
Technique: keep cursor movement inside FileBlock. A concurrent design would instead use positional APIs or lock the cursor.
M2: header validation order
Nudge: Do not use the page size to allocate or multiply until it is validated.
Bigger: Read the fixed 32-byte prefix, check magic/version/page size, then decode counts and compute the expected length with checked u64 arithmetic.
Technique: small field codec helpers using from_be_bytes/to_be_bytes; no struct casts.
M5–M6: cache bookkeeping after eviction
Nudge: A frame index is stored in two places.
Bigger: Before reusing a victim, preserve dirty bytes, remove the old page number from the table, replace the frame, then insert the new mapping.
Technique: FIFO can choose the smallest loaded_at. Increment the pager's monotonic counter whenever a page is loaded into a frame.
M7: why page count and file length move together
Nudge: A clean header must not name bytes that the file does not contain.
Bigger: Compute the new page number/count with checked arithmetic, extend, write the zero page, then mark the in-memory header dirty.
Technique: sync persists dirty frames and the encoded metadata page before calling the backend synchronization request. This gives clean reopen behavior only; Phase 5 supplies atomic publication.
