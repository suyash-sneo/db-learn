0.4 — Structs, enums, pattern matching

Goal: Model data the Rust way: struct for "has-a" (a page has a number and bytes), enum for "is-one-of" (a node is a leaf or an internal node), and match to act on which variant you've got.

Why this matters to your DB

Almost everything in the engine is a struct or an enum. A Page is a struct. A B+tree Node is an enum (Leaf or Internal). A serialized document field is an enum (string vs int vs nested doc). Pattern matching is how you'll branch on "what kind of node is this" — and Rust forces you to handle every case, which kills a whole class of bugs.

Structs — bundling related data

rust 
struct Page {
    number: u32,
    dirty: bool,
    data: Vec<u8>,
}

let p = Page { number: 1, dirty: false, data: vec![0u8; 4096] };
println!("page {} has {} bytes", p.number, p.data.len());

Methods with impl

Behavior lives in an impl block. The first parameter is the receiver: &self (read), &mut self (mutate), or self (consume). A function without self is an associated function — like a static constructor; new by convention.

rust 
impl Page {
    // associated function (constructor); call as Page::new(1)
    fn new(number: u32) -> Page {
        Page { number, dirty: false, data: vec![0u8; 4096] }
    }
    // method, reads self
    fn size(&self) -> usize { self.data.len() }
    // method, mutates self
    fn mark_dirty(&mut self) { self.dirty = true; }
}

let mut p = Page::new(1);
p.mark_dirty();
println!("{}", p.size());

Note number, alone in new: when a field and a local variable share a name, Rust lets you write the shorthand instead of number: number.

Enums — "is one of these"

This is where Rust outshines most languages. An enum variant can carry data, and each variant can carry different data. This is a "sum type" / "tagged union."

rust 
enum Node {
    Leaf { keys: Vec<i64>, values: Vec<Vec<u8>> },
    Internal { keys: Vec<i64>, children: Vec<u32> },
}

A Node value is either a Leaf or an Internal, never both, and the compiler knows which. That's exactly a B+tree node. You'll also meet the two most important enums in the standard library next lesson: Option<T> (Some/None) and Result<T,E> (Ok/Err) — both are just enums.

Pattern matching with match

match branches on a value and destructures it — pulls the inner data out — in one step. Critically, it must be exhaustive: you handle every variant or the code won't compile. Add a new node type later and the compiler points you at every match you forgot to update. [src: Rust Book: match]

rust 
fn key_count(node: &Node) -> usize {
    match node {
        Node::Leaf { keys, .. }     => keys.len(),
        Node::Internal { keys, .. } => keys.len(),
    }
}

The .. means "ignore the other fields." Each arm is pattern => expression. match is an expression, so it produces a value you can assign or return.

Matching Option — the everyday case

rust 
fn describe(maybe: Option<i64>) -> String {
    match maybe {
        Some(n) if n > 0 => format!("positive: {}", n), // guard
        Some(n)          => format!("non-positive: {}", n),
        None             => "nothing".to_string(),
    }
}

The if n > 0 is a match guard — an extra condition on an arm. Patterns can also bind (Some(n) binds the inner value to n).

Shorthands: if let and while let

When you only care about one variant, match is verbose. if let is the shortcut:

rust 
if let Some(n) = maybe {
    println!("got {}", n);
} else {
    println!("empty");
}
🦀 Rust just-in-time
  • #[derive(Debug)] above a struct/enum auto-generates a printable form; then println!("{:?}", x) works. #[derive(Clone)] gives you .clone(). You'll derive these constantly.
  • Tuple structs exist too: struct PageNo(u32); — a named wrapper around one value.
  • An enum variant with no data is a plain tag: enum Color { Red, Green }.

✅ Concept check

1. When do you reach for an enum instead of a struct?

Show answer
When a value is one of several alternatives (leaf vs internal node, string vs int field). A struct is for a value that has all of several fields at once.

2. What does "match must be exhaustive" buy you when you add a third node variant later?

Show answer
The compiler errors on every match that doesn't handle the new variant, so you can't silently forget a case. Refactors become safe.

3. What's the difference between &self, &mut self, and self as a method receiver?

Show answer
&self borrows to read, &mut self borrows to mutate, self takes ownership (consumes the value — useful for "convert into" methods).

4. Rewrite this match on an Option that only acts on Some using a shorter form.

Show answer
Use if let Some(x) = opt { … }.
