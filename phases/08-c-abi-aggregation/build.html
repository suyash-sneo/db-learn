Build: a real C ABI + a Dart binding

Goal: Wrap your Phase 6 document store in a stable, panic-safe C ABI (opaque handle, byte buffers, error codes, cursor, a small aggregation pipeline), generate a C header with cbindgen, prove it with a C smoke test, and call it from a minimal Dart binding.
⚠️ AI off

This page gives the ABI contract, milestones, failing tests, and todo!() stubs only — no worked boundary logic. The FFI boundary is where memory-safety bugs become process crashes; implement the pointer handling, ownership, panic wrapping, and cursor logic yourself, following the rules in 8.1–8.5.

⚠️ What the in-process tests do and do not prove

The Rust tests below call the extern "C" functions in the same process through raw pointers. They validate the boundary logic — null/empty handling, ownership round-trips, error codes, cursor iteration. They do not prove cross-language ABI correctness, header accuracy, symbol export, or on-device loading. Those require the mandatory manual checks in the acceptance criteria: generate the header with --lang c, inspect it, compile and run a C smoke test against the built cdylib, and round-trip one call from Dart.

Deliverable

A Rust 2024 crate minidb_capi with crate-type = ["cdylib", "rlib"], wrapping the Phase 6 docstore (and optionally the Phase 7 lockmgr). It exposes opaque Db and Cursor handles and a small set of prefixed, unmangled functions. Every exported function is wrapped in catch_unwind and returns an i32 status. A cbindgen.toml with language = "C" generates minidb.h; a C smoke test and a Dart binding consume it.

The ABI surface

FunctionContract
minidb_abi_version() -> u32Non-zero ABI version; bump on incompatible change.
minidb_open(path, *mut *mut Db) -> i32Allocates a handle; caller must minidb_close it.
minidb_close(*mut Db)Frees the handle; null is a no-op.
minidb_insert(db, *const u8, usize, *mut i64) -> i32Borrowed doc bytes; writes new _id.
minidb_find(db, *const u8, usize, *mut *mut u8, *mut usize) -> i32Engine-allocated result bytes; free with minidb_free_buf.
minidb_aggregate(db, *const u8, usize, *mut *mut u8, *mut usize) -> i32Serialized pipeline in, serialized docs out.
minidb_find_cursor / _cursor_next / _cursor_closeStreaming results (8.3); cursor valid only while its Db is open.
minidb_free_buf(*mut u8, usize)Frees an engine buffer using the exact (ptr,len) issued.
minidb_last_error() -> *const c_charThread-local detail for the most recent failure.

Ordered milestones

  1. M1 — handle lifecycle. minidb_open/minidb_close with correct Box ownership; minidb_abi_version.
  2. M2 — byte-buffer CRUD. minidb_insert (borrowed input, the (NULL,0)/(NULL,nonzero) rules) and minidb_find + matching minidb_free_buf (the Box<[u8]> round-trip; empty = (NULL,0)).
  3. M3 — error + panic safety. Stable error codes; wrap every export in catch_unwind; populate a thread-local minidb_last_error; ensure a caught panic leaves committed state intact (transactional mutation, Phase 5).
  4. M4 — cursor. find_cursor/cursor_next (three-valued return)/cursor_close, with the documented lifetime & invalidation policy.
  5. M5 — aggregation subset. minidb_aggregate running a small stage chain ($match via your evaluator, plus $sort/$limit), documented as a subset.
  6. M6 — header + bindings. cbindgen (--lang c) → inspect → C smoke test → minimal Dart binding that round-trips one open/insert/find.

Acceptance criteria

  • All supplied Rust tests pass unchanged.
  • Inputs honor the borrowed-buffer contract; (NULL,0) is empty, (NULL,nonzero) is rejected.
  • Output buffers round-trip exactly and are freed only via minidb_free_buf.
  • No code path can let a panic unwind past extern "C"; a forced internal panic returns MINIDB_ERR_PANIC and leaves committed data intact.
  • A cursor iterates all matches then signals end; using it after its Db is closed is documented as forbidden.
  • Mandatory manual: cbindgen --lang c produces minidb.h; you inspected it (opaque handles forward-declared, buffer params correct); a C smoke test compiles, links the cdylib, and round-trips a call.
  • Mandatory manual: a Dart program DynamicLibrary.opens the library, looks up the symbols, and round-trips open/insert/find; freeing uses minidb_free_buf.
  • Note the platform caveats (8.7): library naming, signing, ABI coverage — state what you did and did not test on real devices.

fn guard<F: FnOnce() -> i32 + std::panic::UnwindSafe>(f: F) -> i32 {
    match catch_unwind(f) {
        Ok(code) => code,
        Err(_) => { /* set last_error */ MINIDB_ERR_PANIC }

Failing tests

tests/capi.rs — call the extern fns in-process (boundary logic only) 
use minidb_capi::*;
use std::ptr;

#[test]
fn abi_version_is_nonzero() {
    assert!(minidb_abi_version() > 0);
}

#[test]
fn open_insert_find_roundtrip() {
    let mut db: *mut Db = ptr::null_mut();
    let path = std::ffi::CString::new(":memory:").unwrap();
    assert_eq!(minidb_open(path.as_ptr(), &mut db), MINIDB_OK);

    let doc = br#"{"city":"London"}"#;
    let mut id: i64 = 0;
    assert_eq!(minidb_insert(db, doc.as_ptr(), doc.len(), &mut id), MINIDB_OK);
    assert!(id > 0);

    let filter = br#"{"city":"London"}"#;
    let mut out: *mut u8 = ptr::null_mut();
    let mut out_len: usize = 0;
    assert_eq!(minidb_find(db, filter.as_ptr(), filter.len(), &mut out, &mut out_len), MINIDB_OK);
    assert!(out_len > 0 && !out.is_null());
    minidb_free_buf(out, out_len);
    minidb_close(db);
}

#[test]
fn null_nonzero_input_is_rejected() {
    let mut db: *mut Db = ptr::null_mut();
    let path = std::ffi::CString::new(":memory:").unwrap();
    minidb_open(path.as_ptr(), &mut db);
    let mut id: i64 = 0;
    // (NULL, nonzero) must not be treated as a valid empty buffer
    assert_ne!(minidb_insert(db, ptr::null(), 5, &mut id), MINIDB_OK);
    minidb_close(db);
}

#[test]
fn free_buf_handles_null_and_empty() {
    minidb_free_buf(ptr::null_mut(), 0); // no-op, must not crash
}

#[test]
fn forced_panic_becomes_error_not_abort() {

    // MINIDB_ERR_PANIC rather than aborting the process.
    assert_eq!(minidb_panic_for_test(), MINIDB_ERR_PANIC);
}

Stuck? — graduated hints

Box ownership for the handle confuses me
Nudge: open leaks, close reclaims.
Bigger: Box::into_raw(Box::new(db)) gives the caller a pointer; Box::from_raw in close turns it back into an owned Box that drops.
Technique: never hand out a pointer to a stack value or a &mut; the handle must outlive the call, so it must be heap-owned via Box.
The output buffer crashes on free
Nudge: The free must see the same length you issued.
Technique: build the buffer with Vec::into_boxed_slice + Box::into_raw, and reclaim with slice_from_raw_parts_mut(ptr, len) + Box::from_raw using the identical len. A mismatched length is UB.
How do I test panic safety without aborting?
Nudge: Only an unwinding panic that reaches extern "C" aborts.
Technique: wrap the body in catch_unwind so the panic is caught and mapped to a code; add a #[cfg(test)] function that deliberately panics inside the guard and assert it returns MINIDB_ERR_PANIC. (Don't build with panic = "abort", which catch_unwind can't catch.)
cbindgen emitted C++ / weird types
Nudge: Default language is C++.
Technique: set language = "C" in cbindgen.toml (or --lang c), regenerate, and read the header: opaque handles should be forward-declared structs, buffers should be uint8_t* + size_t. Then compile a C file that includes it and links the cdylib.
