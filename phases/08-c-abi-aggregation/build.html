Build: the real C ABI + Dart binding

Goal: Wrap the Phase 6 document store in a versioned C ABI with complete CRUD, engine-owned output buffers, diagnostics, a materialized-result cursor, a generated C header, and a Dart binding. Aggregation is an optional extension after the core boundary is proven.
⚠️ AI off

This page contains contracts, milestones, acceptance criteria, compile-ready declarations, todo!() stubs, failing tests, and graduated hints. It deliberately contains no working handle, buffer, panic-guard, cursor, CRUD, or binding implementation.

⚠️ Three different things must be tested
  • Rust boundary tests exercise detectable null/empty cases, status codes, ownership round-trips, CRUD, poisoning, and cursor state in-process.
  • A compiled C smoke test proves that the generated header, exported symbols, calling convention, linker inputs, and runtime library agree.
  • A Dart/package test proves generated Dart types, native allocation cleanup, code-asset resolution, and the packaged mobile artifact. Rust tests cannot substitute for either external consumer.

Deliverable and fixed policy

Create a Rust 2024 crate named minidb_capi with crate-type = ["cdylib", "rlib"]. It wraps the Phase 6 store without exposing core Rust types. These choices are fixed for the exercise:

  • One Db handle is single-caller and non-reentrant.
  • Paths are borrowed UTF-8 byte pairs, not NUL-terminated strings.
  • Documents, filters, and result arrays use the Phase 6 JSON codec.
  • (NULL, 0) is accepted as empty input; (NULL, nonzero) is MINIDB_ERR_NULL_ARG. An empty filter means “match all”; an empty document is parsed and therefore returns MINIDB_ERR_BAD_DOC.
  • All engine outputs are Box<[u8]>-backed exact pointer/length pairs; empty output is (NULL, 0).
  • The v1 cursor owns a materialized result set and shared ownership of the engine/session, so closing the original Db handle does not dangle the cursor. It does not claim lazy query execution.
  • A caught unwinding panic poisons the handle. Normal calls then return MINIDB_ERR_POISONED; diagnostics and close remain available.

ABI surface

FunctionContract

Ordered milestones

  1. M1 — crate + lifecycle. Crate types, opaque non-zero-sized Rust stubs, ABI version, UTF-8 path validation, open/close ownership, and safe-default out-parameters.
  2. M2 — complete CRUD. Borrowed input helper; insert/find/update/delete; stable status mapping; find no-matches returns a successful serialized empty array.
  3. M3 — output ownership. Exact boxed-slice export/free, null/zero normalization, Dart copy-before-free, repeated lifecycle stress, and an external leak-aware run.
  4. M4 — panic + diagnostics. Outermost unwind guard, per-handle detail, forced-panic feature hook, poisoning, and reopen-from-committed-state manual verification.
  5. M5 — cursor. Materialized result ownership, stable END behavior, cursor survives closing the original database handle, and exact per-document frees.
  6. M6 — generated consumers. cbindgen 0.29.4 C header, C11 smoke test, ffigen 20.1.1 Dart binding, and desktop/dev round-trip.
  7. M7 — mobile package. Flutter 3.44 package_ffi hook invokes the Rust build, registers the code asset, and passes the artifact/device checks from 8.7.
  8. Optional M8 — aggregation. Only after M1–M7: $match, $sort, and $limit with exact documented semantics and limits.

Acceptance criteria

  • All supplied tests compile unchanged with cargo test --features ffi-test-hooks --no-run, then pass after implementation.
  • Every testable acceptance criterion below has a supplied test; inherently unsafe misuse such as a fabricated pointer, wrong non-null allocation length, double free, or use-after-close is documented as caller UB and is never invoked by tests.
  • All output parameters are safe defaults on every error. Empty and null/nonzero inputs are distinguished exactly as specified.
  • CRUD includes update and delete. Missing update/delete IDs return NOT_FOUND; find with no matches is successful.
  • A forced unwinding panic returns PANIC, poisons the handle, and does not escape the C ABI. Manual: reopen the file and verify only previously committed state is visible.
  • A cursor emits every materialized match exactly once, returns stable END, remains valid after the original database handle closes, and releases every output/cursor allocation.
  • Mandatory external: generated C header is inspected; a C11 program compiles, links, runs, and checks the ABI version plus one lifecycle round-trip.
  • Mandatory external: generated Dart wrapper checks the ABI version, performs open/insert/find/free/close, and releases Dart and engine allocations in finally.
  • Mandatory packaging: inspect packaged symbols/architectures, run Android 16 KB validation when Android 15+ Play support is claimed, and run an iOS release-mode device build when iOS support is claimed.
  • Mandatory leak check: run the repeated lifecycle test under an available leak-aware native tool. The loop alone is not proof of no leaks.

Why the test hook uses a feature

An integration test in tests/ links the library as a normal dependency, so a symbol hidden only behind #[cfg(test)] is not available there. The explicit ffi-test-hooks feature makes the hook available for this test build and keeps it absent from normal release builds.

use minidb_capi::*;
use std::ptr;

unsafe fn open_memory() -> *mut Db {
    let path = b":memory:";
    let mut db = ptr::null_mut();
    let rc = unsafe {
        minidb_open(path.as_ptr(), path.len(), &mut db)
    };
    assert_eq!(rc, MINIDB_OK);
    assert!(!db.is_null());
    db
}

unsafe fn take_output(ptr: *mut u8, len: usize) -> Vec<u8> {
    if len == 0 {
        assert!(ptr.is_null());
        return Vec::new();
    }
    assert!(!ptr.is_null());
    let copied = unsafe {
        std::slice::from_raw_parts(ptr, len).to_vec()
    };
    assert_eq!(unsafe { minidb_free_buf(ptr, len) }, MINIDB_OK);
    copied
}

#[test]
fn cursor_survives_db_handle_close_and_end_is_stable() {
    let db = unsafe { open_memory() };
    for n in 1..=2 {
        let doc = format!(r#"{{"n":{n}}}"#);
        let mut id = 0;
        assert_eq!(unsafe { minidb_insert(db, doc.as_ptr(), doc.len(), &mut id) }, MINIDB_OK);
    }
    let mut cursor = ptr::null_mut();
    assert_eq!(unsafe { minidb_find_cursor(db, ptr::null(), 0, &mut cursor) }, MINIDB_OK);
    assert_eq!(unsafe { minidb_close(db) }, MINIDB_OK);
    let mut seen = 0;
    loop {
        let mut out = ptr::null_mut();
        let mut out_len = 0;
        let rc = unsafe { minidb_cursor_next(cursor, &mut out, &mut out_len) };
        if rc == MINIDB_END {
            assert!(out.is_null());
            assert_eq!(out_len, 0);
            break;
        }
        assert_eq!(rc, MINIDB_OK);
        let _ = unsafe { take_output(out, out_len) };
        seen += 1;
    }
    assert_eq!(seen, 2);
    let mut out = ptr::dangling_mut();
    let mut out_len = 9;
    assert_eq!(unsafe { minidb_cursor_next(cursor, &mut out, &mut out_len) }, MINIDB_END);
    assert!(out.is_null());
    assert_eq!(out_len, 0);
    assert_eq!(unsafe { minidb_cursor_close(cursor) }, MINIDB_OK);
}

Stuck? — graduated hints

How do I structure the pointer checks?
Nudge: Write harmless values to valid output locations before parsing input.
Bigger nudge: Separate three jobs: validate out-pointers, map a borrowed input pair to a call-scoped slice, then publish owned output only after success.
Technique: Keep each raw-pointer operation in a small documented unsafe block. Use slice::from_raw_parts only after handling null/zero and all documented size/lifetime preconditions.
How do I round-trip an output buffer?
Nudge: The allocation shape returned and reclaimed must match exactly.
Bigger nudge: Remove spare Vec capacity before exposing the pair.
Technique: Name the relevant APIs: Vec::into_boxed_slice, Box::into_raw, ptr::slice_from_raw_parts_mut, and Box::from_raw. Do not add a second allocator convention.
catch_unwind rejects my closure or the handle might be damaged
Nudge: Unwind containment and logical recovery are separate concerns.
Bigger nudge: The outer wrapper maps ordinary errors, catches an unwinding panic, records detail, and marks the shell handle poisoned.
Technique: Read catch_unwind and AssertUnwindSafe. Using the latter is an assertion you must justify; poisoning is still required. Explicitly test with unwinding enabled—panic="abort" cannot be caught.
How can a cursor survive Db close?
Nudge: Do not store a naked borrow to an allocation the caller can destroy.
Bigger nudge: Separate the foreign Db handle box from the underlying engine/session lifetime.
Technique: Give both the database handle and cursor shared ownership of an internal owner. The v1 cursor additionally owns its materialized results and index.
Why is my integration-test panic hook missing?
Nudge: Integration tests compile the library as a normal dependency.
Bigger nudge: #[cfg(test)] is not enabled for that dependency build.
Technique: Gate the exported hook behind the declared ffi-test-hooks feature and run the exact command shown above; ensure release packaging does not enable it.
cbindgen emitted a definition or the wrong length type
Nudge: Opaque Rust types should not have a public guaranteed layout.
Bigger nudge: Do not make the placeholder zero-sized; cbindgen documents that zero-sized types can be erased from signatures.
Technique: Keep private non-zero state with no repr(C), set language = "C" and usize_is_size_t = true, regenerate, then inspect and compile the header.
