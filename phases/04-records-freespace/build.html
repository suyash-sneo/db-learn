Build: a slotted record store with overflow and page reuse

Goal: Store variable-length byte records in self-describing record pages, preserve live record identifiers through compaction, spill large tails into checked overflow chains, and persist a typed singly-linked freelist.
⚠️ AI off

This page defines formats and contracts, then gives only todo!() stubs, failing tests, and hints. Slotted-page mutation, overflow, compaction, and freelist logic are prohibited solution components; implement them yourself.

Scope and identity rule

RecordId { page: PageNo, slot: u16 } is an internal physical identifier. It remains valid while that live record is compacted within the same page. After deletion it is invalid; a later insert may reuse the tombstoned slot, so stale identifiers are not safe public handles. Moving a record to another page would require returning/updating a new identifier and is deferred to the transactional document layer.

Exact page formats

All integers are big-endian. Page numbers use Phase 2's u32 PageNo.

Record page

OffsetBytesField
01PAGE_RECORD = 0x21
11reserved, zero
22slot count, including tombstones
42live record count
62cell-content start
82fragmented/dead cell bytes
102reserved, zero

The 12-byte header is followed by 4-byte slots: offset:u16 | length:u16. (0, 0) is a tombstone. Other zero combinations are corruption. Cells grow backward from the page end. Insertion may reuse a tombstone slot; compaction keeps every slot index fixed and changes only live offsets.

Record cell

flags:u8 | total_len:u32 | inline_len:u16 | first_overflow:u32 | inline_bytes. Header size is 11 bytes. Flag bit 0 means an overflow tail exists; all other bits must be zero. inline_len ≤ total_len. If the flag is clear, inline_len == total_len and first_overflow == 0. If set, inline_len < total_len and the first overflow page is a valid data page.

Overflow and free pages

Both have an 8-byte header: type:u8 | reserved:[u8;3] | next:u32. Types are PAGE_OVERFLOW = 0x22 and PAGE_FREE = 0x23. Overflow payload begins at byte 8. The record's total_len determines how many payload bytes are meaningful in the final page. A free page's remaining bytes are zero.

Required validation

  • Every header, slot, cell, and chain range is checked before slicing.
  • Overflow/freelist traversal rejects page 0, page 1, out-of-range pages, wrong page types, cycles, early termination, and a nonzero tail after the requested byte count is complete.
  • Record length is at most u32::MAX. Practical allocation limits may be lower.
  • The complete inline cell plus one 4-byte slot may use at most one quarter of page_size − 12; larger tails overflow.
  • A record page with live_count == 0 may be returned to the freelist after its last record and overflow chain are deleted.

Ordered milestones

  1. M1 — Record page primitives. Initialize, validate, insert, read, tombstone, and reuse slots without changing other live slot indices.
  2. M2 — Compaction. Rebuild a page in scratch, preserving slot indices and tombstones; retry an insert once when total free space is sufficient.
  3. M3 — Typed freelist. Pop/push PAGE_FREE pages using the pager's persisted freelist head; reject double-free and cycles.
  4. M4 — Overflow. Write/read/free typed overflow chains and enforce exact total length.
  5. M5 — Record API. Scan existing record pages for space, allocate/initialize a new one when needed, and implement insert/get/delete/sync/reopen.

Acceptance criteria

  • All supplied tests pass unchanged.
  • Empty, inline, and multi-page records round-trip.
  • Deleting one record does not change another record's identifier.
  • Compaction preserves all live identifiers and permits insertion when total free bytes suffice.
  • Freed record/overflow pages are reused before page_count grows.
  • Malformed and cyclic chains return typed errors rather than hanging or panicking.
  • Sync and reopen preserve live records and the freelist.
  • Manual integration check: repeat reopen and reuse with FileBlock. Atomic insert/delete across several pages is not claimed until Phase 5 transactions.

Failing tests

append as an inline unit-test module in src/recstore.rs 
#[cfg(test)]
mod tests {
    use super::*;
    use pager::MemBlock;

    fn fresh() -> RecStore<MemBlock> {
        let p = Pager::create(MemBlock::empty(), 4096, 8).unwrap();
        RecStore::open(p).unwrap()
    }

    #[test]
    fn empty_small_and_large_records_roundtrip() {
        let mut s = fresh();
        let empty = s.insert(b"").unwrap();
        let small = s.insert(b"hello").unwrap();
        let big = (0..20_000).map(|i| (i % 251) as u8).collect::<Vec<_>>();
        let large = s.insert(&big).unwrap();
        assert!(s.get(empty).unwrap().is_empty());
        assert_eq!(s.get(small).unwrap(), b"hello");
        assert_eq!(s.get(large).unwrap(), big);
    }

    #[test]
    fn delete_does_not_shift_other_record_ids() {
        let mut s = fresh();
        let a = s.insert(b"a").unwrap();
        let b = s.insert(b"b").unwrap();
        s.delete(a).unwrap();
        assert_eq!(s.get(b).unwrap(), b"b");
        assert!(matches!(s.get(a), Err(RecError::NotFound)));
    }

    #[test]
    fn compaction_preserves_live_slot_indices() {
        let mut s = fresh();
        let a = s.insert(&vec![1; 600]).unwrap();
        let dead = s.insert(&vec![2; 600]).unwrap();
        let c = s.insert(&vec![3; 600]).unwrap();
        assert_eq!(a.page, c.page);
        s.delete(dead).unwrap();
        s.compact_page(a.page).unwrap();
        assert_eq!(s.get(a).unwrap(), vec![1; 600]);
        assert_eq!(s.get(c).unwrap(), vec![3; 600]);
    }

    #[test]
    fn churn_reuses_holes_after_compaction() {
        let mut s = fresh();
        let mut ids = Vec::new();
        for i in 0..40 { ids.push(s.insert(&vec![i as u8; 80]).unwrap()); }
        for id in ids.iter().step_by(2) { s.delete(*id).unwrap(); }
        for i in 0..20 { s.insert(&vec![200 + i as u8; 80]).unwrap(); }
    }

    #[test]
    fn freed_pages_are_reused_before_growth() {
        let mut s = fresh();
        let big = vec![7u8; 30_000];
        let id = s.insert(&big).unwrap();
        let high_water = s.pager.page_count();
        s.delete(id).unwrap();
        let id2 = s.insert(&big).unwrap();
        assert_eq!(s.pager.page_count(), high_water);
        assert_eq!(s.get(id2).unwrap(), big);
    }

    #[test]
    fn overflow_cycle_is_rejected() {
        let mut s = fresh();
        let first = s.write_overflow(&vec![9; 5000]).unwrap();
        s.pager.modify_page(first, |page| page[4..8].copy_from_slice(&first.to_be_bytes())).unwrap();
        assert!(matches!(
            s.read_overflow(first, 10_000),
            Err(RecError::OverflowCycle(p)) if p == first
        ));
    }

    #[test]
    fn sync_and_reopen_preserve_records_and_freelist() {
        let mut s = fresh();
        let keep = s.insert(&vec![4; 12_000]).unwrap();
        let remove = s.insert(&vec![5; 12_000]).unwrap();
        s.delete(remove).unwrap();
        let high_water = s.pager.page_count();
        s.sync().unwrap();
        let io = s.into_pager().into_inner();
        let pager = Pager::open(io, 8).unwrap();
        let mut reopened = RecStore::open(pager).unwrap();
        assert_eq!(reopened.get(keep).unwrap(), vec![4; 12_000]);
        reopened.insert(&vec![6; 12_000]).unwrap();
        assert_eq!(reopened.pager.page_count(), high_water);
    }
}

Stuck? — graduated hints

M1: deletion changes another RecordId
Nudge: Slot index is identity on a record page.
Bigger: Do not remove or shift the 4-byte slot. Replace it with (0,0) and update counts.
Technique: insertion may reuse a tombstone index; document that a deleted identifier is invalid and may later alias a different record.
M2: compaction corrupts overlapping source bytes
Nudge: Reading and rewriting the same page in slot order can overwrite a cell not yet copied.
Bigger: Validate first, copy live cells into owned scratch storage, then build a complete new page image.
Technique: preserve the entire slot array shape, including tombstones; only live offsets and cell_start change.
M3: freelist corruption or double allocation
Nudge: Validate the head page before publishing the next head.
Bigger: A free page must have PAGE_FREE, zero reserved bytes, and a next pointer that is 0 or an allocated data-page number.
Technique: use a visited set while validating on open. Read old head → write typed free page → publish new head when freeing; reverse the state change when allocating.
M4: final overflow page returns extra zeros
Nudge: Page capacity is not payload length.
Bigger: Track the exact remaining byte count from the cell's total_len - inline_len.
Technique: consume min(remaining, page_size - 8) per page; when remaining reaches zero, require next == 0.
