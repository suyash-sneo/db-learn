Build: byte-buffer round trip, Rust → C ABI → Dart

Goal: Prove that a Uint8List can enter Rust through a C ABI, be transformed, return in Rust-owned memory, be copied by Dart, and be freed exactly once.
⚠️ AI off

Use the lessons, primary documentation, compiler errors, and the hint ladder. The page supplies contracts, stubs, and failing tests—not the implementation. Boundary code is the skill being tested.

Deliverable

Create a Rust 2024 package named ffi_echo that builds an rlib for tests and a cdylib for Dart, generate a C header, and call it from a one-screen Flutter or Dart desktop app.

Cargo.toml 
[package]
name = "ffi_echo"
version = "0.1.0"
edition = "2024"

[lib]
crate-type = ["rlib", "cdylib"]

The safe transform returns a new vector where every byte is incremented with wrapping arithmetic: 255 becomes 0. The C surface is fixed:

  • echo_transform(in_ptr, in_len, out_ptr, out_len) -> i32
  • echo_free(ptr, len)
  • echo_version() -> i32

Input is caller-owned and borrowed only during the call. Nonempty output is a Rust Box<[u8]> represented by pointer + length. Empty output is (null, 0). The caller copies nonempty output, then returns the unchanged pair to echo_free exactly once.

Ordered milestones

  1. M1 — Build and symbol. Export echo_version; inspect the library.
  2. M2 — Safe core. Implement and unit-test transform.
  3. M3 — Boundary. Validate pointers, handle (null, 0) without from_raw_parts, call the safe core, transfer a boxed slice, and reclaim it in echo_free.
  4. M4 — Panic containment. Implement one boundary guard used by the transform and the test-only panic probe. Use unwind mode for tests; caught panics return ERR_PANIC.
  5. M5 — Header. Generate C—not C++—with cbindgen.
  6. M6 — Dart. Bind the exact header signatures, test nonempty and empty input, and release allocations in finally blocks.

Acceptance criteria

  • All Rust tests below pass.
  • Dart displays [1, 2, 3] → [2, 3, 4] and empty input → empty output.
  • Null input with nonzero length and null out-parameters return ERR_NULL.
  • The test-only forced panic returns ERR_PANIC; it does not abort the test process.
  • A repeated allocate/copy/free test passes. A leak-aware external tool reports no lost allocations; the Rust unit test alone does not prove absence of leaks.
  • The generated header and Dart native typedefs match exactly.

Failing Rust tests

append to src/lib.rs 
#[cfg(test)]
mod tests {
    use super::*;
    use std::ptr::{null, null_mut};

    unsafe fn call(input: &[u8]) -> Vec<u8> {
        let mut ptr = null_mut();
        let mut len = 0usize;
        let in_ptr = if input.is_empty() { null() } else { input.as_ptr() };
        let rc = unsafe {
            echo_transform(in_ptr, input.len(), &mut ptr, &mut len)
        };
        assert_eq!(rc, OK);

        let result = if len == 0 {
            assert!(ptr.is_null());
            Vec::new()
        } else {
            assert!(!ptr.is_null());
            unsafe { std::slice::from_raw_parts(ptr, len) }.to_vec()
        };
        unsafe { echo_free(ptr, len) };
        result
    }

    #[test]
    fn core_transform() {
        assert_eq!(transform(&[1, 2, 255]), vec![2, 3, 0]);
        assert!(transform(&[]).is_empty());
    }

    #[test]
    fn boundary_nonempty_and_empty() {
        assert_eq!(unsafe { call(&[10, 20, 30]) }, vec![11, 21, 31]);
        assert!(unsafe { call(&[]) }.is_empty());
    }

    #[test]
    fn rejects_invalid_nulls() {
        let mut ptr = null_mut();
        let mut len = 0usize;
        assert_eq!(
            unsafe { echo_transform(null(), 1, &mut ptr, &mut len) },
            ERR_NULL,
        );
        let input = [1u8];
        assert_eq!(
            unsafe { echo_transform(input.as_ptr(), 1, null_mut(), &mut len) },
            ERR_NULL,
        );
    }

    #[test]
    fn panic_is_contained() {
        assert_eq!(panic_probe(), ERR_PANIC);
    }

    #[test]
    fn repeated_lifecycle() {
        let input = [7u8; 1024];
        for _ in 0..10_000 {
            let output = unsafe { call(&input) };
            assert_eq!(output.len(), input.len());
            assert_eq!(output[0], 8);
        }
    }
}

Dart smoke-test checklist

  • Bind from the generated header manually or with ffigen.
  • Use Size for size_t and a pointer-to-pointer for uint8_t **out_ptr.
  • Pass (nullptr, 0) for empty input.
  • Copy Rust output with Uint8List.fromList before echo_free.
  • Put Rust-output cleanup in an inner finally and Dart-allocation cleanup in an outer finally.
  • Show both nonempty and empty results in the app.

Leak validation

repeated_lifecycle catches crashes, wrong lengths, and many double-free errors, but its assertions cannot prove that memory was returned. Run the test binary or a small C harness under a leak-aware tool available on your platform (for example Valgrind on supported Linux systems or a sanitizer-enabled build), and record the exact command and clean report in the project README.

Stuck? — graduated hints

M3: empty input crashes in from_raw_parts
Nudge: Rust's slice constructor does not accept null, even for length zero.
Bigger: Branch on in_len == 0 before constructing a slice.
Technique: map the valid ABI pair (null, 0) directly to &[]; reject null with nonzero length.
M3: how do I transfer output without capacity?
Nudge: A vector has capacity, but a boxed slice has exactly a length.
Bigger: Convert with into_boxed_slice before transferring ownership.
Technique: use Box::into_raw; in the free function rebuild a raw slice pointer from pointer + length and reclaim it with Box::from_raw.
M4: catch_unwind rejects my closure
Nudge: The compiler is asking whether observed state is safe after a partial unwind.
Bigger: Keep all output publication until after the transform succeeds.
Technique: use AssertUnwindSafe only after you can explain why no partially published allocation or invalid handle state remains.
M5: cbindgen produced C++
Nudge: Check the tool's default output language.
Technique: pass --lang c or set language = "C" in cbindgen.toml.
M6: Dart crashes or returns garbage
Nudge: Treat the generated C header as the authority.
Bigger: Check Size, every pointer nesting level, and output cleanup.
Technique: bind uint8_t ** as Pointer<Pointer<Uint8>>, copy before freeing, and never pass Rust output to Dart's allocator.
