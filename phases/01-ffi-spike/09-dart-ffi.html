1.9 — dart:ffi: bind, call, copy, and free

Goal: Load the Rust library on desktop, bind the pointer-and-length API, round-trip a Uint8List, and release every native allocation on success and failure.

Why this matters to your DB

This is the first consumer of the stable C surface. Dart never needs to understand Rust Vec, Box, or database internals; it only sees symbols, C-compatible integers, pointers, and documented ownership rules.

1. Dependencies and imports

pubspec.yaml 
dependencies:
  ffi: ^2.1.0
dart 
import 'dart:ffi';
import 'dart:io' show Platform;
import 'dart:typed_data';
import 'package:ffi/ffi.dart';

dart:ffi supplies native types and dynamic-library calls. package:ffi supplies allocators such as calloc. Pin the actual dependency version selected by your project rather than copying a floating “latest” number.

2. Load the desktop development library

dart 
DynamicLibrary openMyDb() {
  if (Platform.isMacOS) {
    return DynamicLibrary.open('target/debug/libmydb.dylib');
  }
  if (Platform.isLinux) {
    return DynamicLibrary.open('target/debug/libmydb.so');
  }
  if (Platform.isWindows) {
    return DynamicLibrary.open(r'target\debug\mydb.dll');
  }
  throw UnsupportedError('Desktop spike only');
}

Those paths are development examples relative to the process working directory. Production Flutter packaging is platform-specific: modern Flutter native assets use build hooks; Android commonly packages a dynamic .so, while Apple platforms may expose statically linked symbols through the process/executable or package a dynamic framework. Do not guess one loading expression for every mobile platform. [src: Flutter native-library linking]

3. Bind the exact C signatures

Assume the generated header declares:

C 
int32_t echo_transform(
    const uint8_t *input,
    size_t input_len,
    uint8_t **out_ptr,
    size_t *out_len);

void echo_free(uint8_t *ptr, size_t len);

Dart needs a native signature and a callable Dart signature. Size maps C size_t and becomes Dart int in the callable form. [src: dart:ffi Size]

dart 
typedef EchoNative = Int32 Function(
  Pointer<Uint8>,
  Size,
  Pointer<Pointer<Uint8>>,
  Pointer<Size>,
);
typedef EchoDart = int Function(
  Pointer<Uint8>,
  int,
  Pointer<Pointer<Uint8>>,
  Pointer<Size>,
);

typedef FreeNative = Void Function(Pointer<Uint8>, Size);
typedef FreeDart = void Function(Pointer<Uint8>, int);

final lib = openMyDb();
final echo = lib.lookupFunction<EchoNative, EchoDart>(
  'echo_transform',
);
final freeEcho = lib.lookupFunction<FreeNative, FreeDart>(
  'echo_free',
);
The generated C header is the signature authority

Do not infer widths from Rust source or from memory. Compare every Dart native typedef with the generated C declaration. A mismatch can compile and then corrupt registers, stack state, or memory. package:ffigen can generate these bindings from the header once you understand the manual form. [src: Dart C interop and ffigen]

4. Complete ownership-safe round trip

dart 
Uint8List echoBytes(Uint8List input) {
  // Represent empty input as the ABI's allowed (nullptr, 0) pair.
  final Pointer<Uint8> inPtr =
      input.isEmpty ? nullptr : calloc<Uint8>(input.length);
  if (input.isNotEmpty) {
    inPtr.asTypedList(input.length).setAll(0, input);
  }

  final outPtr = calloc<Pointer<Uint8>>();
  final outLen = calloc<Size>();

  try {
    final rc = echo(inPtr, input.length, outPtr, outLen);
    if (rc != 0) {
      throw StateError('echo_transform failed: $rc');
    }

    final rustPtr = outPtr.value;
    final length = outLen.value;
    if (rustPtr == nullptr && length != 0) {
      throw StateError('library returned null with nonzero length');
    }

    try {
      if (length == 0) return Uint8List(0);
      // asTypedList is a view; fromList makes the Dart-owned copy.
      return Uint8List.fromList(
        rustPtr.asTypedList(length),
      );
    } finally {
      // Return Rust's allocation even if the Dart copy throws.
      if (rustPtr != nullptr) freeEcho(rustPtr, length);
    }
  } finally {
    // Dart frees only allocations made by calloc.
    if (inPtr != nullptr) calloc.free(inPtr);
    calloc.free(outPtr);
    calloc.free(outLen);
  }
}

Trace the lifetimes

  1. Dart allocates input and out-parameter slots.
  2. Rust borrows input only for the call.
  3. On success, Rust transfers a boxed-slice allocation as outPtr/outLen.
  4. Dart copies the Rust bytes into a new Uint8List.
  5. Dart calls echo_free exactly once for the Rust pair.
  6. The outer finally frees Dart's three native allocations.

Pointer.asTypedList creates a view over native memory; that view must not survive the matching free. The explicit Uint8List.fromList copy is what makes the return value independently Dart-owned.

🦀 Dart FFI just-in-time
  • calloc<T>() allocates one zero-initialized T; passing a count allocates an array.
  • Pointer<Pointer<Uint8>> represents C's uint8_t **.
  • outPtr.value reads the pointer Rust wrote; outLen.value reads the length.
  • Keep long database calls off the UI isolate. This tiny synchronous spike is only proving the ABI path.
✅ Concept check

1. Why are there two Dart typedefs per native function?

Show answer
One describes the exact native C ABI types; the other describes the Dart-callable representation.

2. Why is output freeing in an inner finally?

Show answer
Once Rust returns a live allocation, Dart must return it even if copying or later validation throws.

3. How is empty input represented?

Show answer
As (nullptr, 0), matching the Rust boundary's explicit empty-buffer branch.

4. Why not return rustPtr.asTypedList(length) directly?

Show answer
It is only a view into Rust-owned memory and becomes invalid after echo_free. Return a Dart-owned copy.
