Sources & annotated bibliography

Lessons are self-contained explanations. These links are evidence and optional deeper reading, not missing chapters the learner must leave the site to understand. Web sources on this page were opened and checked on June 20, 2026.

Source policy

Use primary specifications, official documentation, and upstream repositories for factual claims. Books and lectures provide conceptual depth but are not reproduced. Fast-moving tools are pinned in real projects; this bibliography records the upstream location rather than promising that an unpinned “latest” API will stay unchanged.

Rust language, Cargo, and standard library

  • The Rust Programming Language — beginner explanations of ownership, traits, packages, error handling, unsafe Rust, and tests.
  • The Cargo Book and cargo new — package layout, targets, profiles, and current edition selection.
  • Rust 2024 unsafe attributes — why no_mangle is written #[unsafe(no_mangle)].
  • Rust Reference: external blocks and ABIs and type layout.
  • Rustonomicon FFI — C ABI declarations, callbacks, linking, and current unwind behavior.
  • Relevant standard-library contracts: slice::from_raw_parts, Box, catch_unwind, and File::sync_all.
  • Rustlings — short practice exercises.

Serialization

  • Serde guide — framework data model, derive, and attributes.
  • serde_json::to_vec, from_slice, and Value.
  • BSON specification and MessagePack specification — candidates to compare in the document phase.

C ABI, header generation, and Dart/Flutter

  • cbindgen upstream and its CLI/config reference. The CLI defaults to C++; this curriculum explicitly requests C.
  • Dart C interop guide — dart:ffi concepts and package:ffigen for generating Dart bindings from C headers.
  • Dart isolates and concurrency model — separate memory, message passing, and Isolate.run.
  • Flutter — developing packages & plugins — FFI plugins, per-platform native build (CMake/Gradle, CocoaPods), and bundling. Packaging mechanics are version-sensitive.
  • dart:ffi API, Size, and package:ffi.
  • Flutter native library linking — dynamic/static loading distinctions and the current recommendation to use native-assets build hooks for new packages.
  • flutter_rust_bridge v2 docs and upstream repository.

SQLite — storage teacher

SQLite documentation and source are the primary storage references. SQLite states that its source code is in the public domain; still cite exact pages/functions and keep snippets short.

  • Architecture — major modules and layering.
  • Database file format — authoritative byte-level format information.
  • Atomic commit — rollback-journal protocol and filesystem assumptions.
  • Write-ahead logging and WAL format.
  • Locking v3.
  • C interface introduction, C API reference, result codes, and memory-allocation routines.
  • Source repository browser. Planned source dives should identify exact files and functions when authored, including pager.c, btree.c, wal.c, and platform VFS implementations.

Document and embedded-engine references

  • Isar upstream repository — Apache-2.0. Its current README warns that v4 is not ready for production and directs production use to stable v3. Treat branch/version architecture separately; do not blend v3 and v4 claims.
  • DeepWiki Isar index — useful secondary navigation, not a substitute for repository source when making exact architecture claims.
  • LMDB documentation and libmdbx documentation — references for embedded B-tree/MVCC designs. Later lessons must cite the exact version and source file they discuss.
  • MongoDB data modeling, documents, CRUD operations, and the aggregation pipeline.

Concurrency, isolation, and transactions

  • PostgreSQL — Transaction Isolation and MVCC introduction — precise SQL-standard isolation levels, phenomena, and an MVCC implementation.
  • SQLite locking v3 (the five lock states) and WAL / isolation for snapshot-style concurrency.
  • Rust Book ch. 16 — Fearless Concurrency, std::sync (Arc, Mutex, RwLock, Condvar), and the Rustonomicon Send/Sync chapter.

Conceptual backbone

  • Database Internals — Alex Petrov, O'Reilly, 2019, ISBN 978-1492040347. Storage structures and engine architecture; no prose reproduced.
  • Designing Data-Intensive Applications — Martin Kleppmann, O'Reilly, 2017, ISBN 978-1449373320. Storage and retrieval trade-offs; no prose reproduced.
  • CMU 15-445/645 Intro to Database Systems — current course site and lecture materials. Cite a specific offering/lecture if a later lesson depends on ordering or timestamps.
  • Let's Build a Simple Database — a C-based, pedagogical database implementation used only as a hands-on comparison.

If an upstream page moves or a fast-moving project changes behavior, add an exact outstanding item to Verification; do not leave a vague “verify everything” marker in a lesson.
