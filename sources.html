Sources & annotated bibliography

These are the books, courses, docs, and repositories the curriculum is built from. Lessons teach the same ideas in original words; come here when a Go deeper / verify box sends you out. Links marked ⚠️VERIFY were not fetched at authoring time and should be confirmed during your review pass.

Legend

✅ = URL fetched and confirmed live while writing this page. ⚠️VERIFY = plausible but unconfirmed; check it. Licensing notes tell you what may be quoted.

Conceptual backbone

  • Database Internals — Alex Petrov (O'Reilly, 2019). The spine of the storage half. Cited by chapter; no prose reproduced. ISBN 978-1492040347.
  • CMU 15-445/645 — Intro to Database Systems — Andy Pavlo. Free lectures on YouTube + course site 15445.courses.cs.cmu.edu ⚠️VERIFY. Cited by lecture title; any timestamps carry ⚠️VERIFY.
  • Designing Data-Intensive Applications — Martin Kleppmann (O'Reilly, 2017). Chapter 3 ("Storage and Retrieval") for B-tree vs LSM. ISBN 978-1449373320.

SQLite — docs & source (public domain)

SQLite's source is in the public domain, so illustrative snippets may be shown freely. These are the storage-track teacher.

  • ✅ Architecture overview — sqlite.org/arch.html. Names the modules: Interface, Tokenizer, Parser, Code Generator, Bytecode Engine (VM), B-Tree, Pager (Page Cache), OS Interface (VFS), Utilities, Test Code.
  • ✅ File format — fileformat2.html. Confirmed: header is the first 100 bytes (§1.3); magic string "SQLite format 3\0" (16 bytes) at offset 0; page size a power of two 512–65536; B-tree pages in §1.6.
  • ✅ Write-ahead logging — wal.html. Confirmed: append changes to a -wal file, commit = special frame; default auto-checkpoint near 1000 pages; PASSIVE/FULL/RESTART checkpoints.
  • Atomic commit — atomiccommit.html ⚠️VERIFY (rollback-journal scheme; the original durability design).
  • Locking & concurrency — lockingv3.html ⚠️VERIFY (lock states: UNLOCKED, SHARED, RESERVED, PENDING, EXCLUSIVE).
  • C-interface reference — c3ref/intro.html ⚠️VERIFY and the header cintro.html ⚠️VERIFY.
  • WAL file format details — walformat.html ⚠️VERIFY (frame headers, the 32-byte WAL header — confirm exact sizes here).
  • Source files to dive into (the lower / storage track): pager.c, wal.c, btree.c + btreeInt.h, os_unix.c, build.c, main.c, and the amalgamation sqlite3.c / public header sqlite3.h. Browse at sqlite.org/src ⚠️VERIFY.

Isar — Rust embedded NoSQL DB (Apache-2.0)

The upper / document-track reference: schema, collections, indexes, query engine, and the Rust↔Dart FFI boundary. Apache-2.0: short, attributed snippets only.

  • Repo — github.com/isar/isar ⚠️VERIFY.
  • Architecture wiki — deepwiki.com/isar/isar ⚠️VERIFY (Core Architecture / Cross-Platform / Development sections).
  • ⚠️VERIFY Isar v3's core sat on LMDB/libmdbx (an embedded B-tree KV store) rather than a from-scratch engine; v4 reworked the core — confirm the current state and whether the active community fork differs.
