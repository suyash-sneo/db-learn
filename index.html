Self-authored learning track

Build your own database — from the bytes up

A phone-readable curriculum that teaches database internals well enough for you to write a schemaless, single-file, embedded database in Rust — with a C ABI and Flutter/Dart FFI as the first consumer. SQLite and Isar are the teachers; you write every line of the engine.

📐 The one rule that makes this work

Every lesson teaches the concept. Every Build page gives you a spec, failing tests, and todo!() stubs — never a working implementation of a database component. You use AI as a tutor, never an author. That constraint is the whole point: it is how the knowledge actually lands.

What you'll have built by the end

A three-layer system. The diagram below is the map you'll return to after every phase — each phase fills in one part of it.

How to use this

  • Read in order. Phases are an ordering, not a schedule — there are no day/week estimates. Each lesson is ~10–20 minutes and stands alone.
  • Do the Builds AI-off. When you reach a build.html, close the AI. Implement against the failing tests. Use the graduated Stuck? hints, which nudge but never hand you the answer.
  • Trust, but verify. Precise claims carry an inline [src: …] pointer. Anything reconstructed from memory wears a ⚠️VERIFY tag. The Verification page collects every one for your ChatGPT review pass.
  • Tap ☰ (top-left) for the full map; ☀️/🌙 toggles light/dark.

📚 Where this comes from

The coverage and ordering follow Database Internals (Petrov), CMU 15-445 (Pavlo), DDIA (Kleppmann), the SQLite docs & public-domain source, the Isar repo, and the Rust/FFI docs. Lessons are written in original words; the Sources page tells you exactly where to read or watch the same material.

Layers, decisions, and constraints are fixed by design (single-file, B+tree core, BSON/MessagePack buffers across FFI). The curriculum teaches toward them.
