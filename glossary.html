Glossary

Plain-language definitions, in the order you'll meet them. This list grows as new phases are authored. Type to filter.

ABI (Application Binary Interface)
The binary-level contract for how compiled code calls compiled code: how arguments are laid out in registers/stack, how names are mangled, how structs are packed. An API is the source-level contract; the ABI is what survives after compilation. Stable C ABIs are why a Rust library can be called from Dart, Python, or Swift.
Access path / residual filter
A query plan's two parts: the access path generates candidate documents (a full scan or an index scan), and the residual filter is the part of the predicate the access path didn't enforce, applied by the evaluator to each candidate. Every plan must return the same set as a full scan.
ACID
Atomicity, Consistency, Isolation, Durability — the four guarantees a transactional store aims for. At the storage layer you mostly implement atomicity (all-or-nothing) and durability (committed data survives a crash).
Aggregation pipeline
A sequence of stages (match → group → sort → …) that transform a stream of documents into a computed result. MongoDB's model; a v2 feature for your engine.
Borrow / borrowing
In Rust, accessing a value through a reference (&T or &mut T) without taking ownership. The borrow checker enforces: many shared & borrows OR exactly one &mut, never both at once.
BSON
Binary JSON — MongoDB's document serialization format. Length-prefixed, typed, supports more types than JSON (dates, binary, ObjectId). A candidate for documents crossing your FFI boundary as byte buffers. [src: BSON specification]
B-tree
A balanced, high-fan-out search tree designed for block storage: each node fills a page, so a shallow tree means few disk reads. The classic database index.
B+tree
A B-tree variant where all values live in the leaf nodes and internal nodes hold only keys for routing; leaves are often linked for fast range scans. The core index you'll build.
Buffer pool / page cache
An in-memory cache of pages read from the file, so repeated access avoids disk I/O. Tracks which pages are dirty (modified, must be written back) and which are pinned (in use, must not be evicted).
catch_unwind
Rust's std::panic::catch_unwind catches an unwinding Rust panic. At a normal extern "C" boundary, an escaping panic aborts the process; catching it first lets an embedded library return an error code instead. It cannot catch panic="abort" or arbitrary crashes. [src: std docs]
cbindgen
A tool that reads public Rust declarations and generates a C or C++ header. Its CLI defaults to C++, so this project explicitly selects C with --lang c or language = "C".
Cell
SQLite's term for a single record stored on a B-tree page (key + payload, possibly with an overflow pointer). Located via the page's cell-pointer array. [src: SQLite file format: B-tree pages]
Checkpoint
Moving committed changes from the WAL back into the main database file, then (eventually) resetting or reusing the WAL. Balances read speed against WAL growth. [src: SQLite WAL checkpointing]
Collection
A named group of documents, implemented as a record heap (the document bytes) plus a primary index mapping each _id to the record's location, plus any secondary indexes. The document-database analogue of a table.
Copy-on-write (CoW)
Instead of overwriting a page in place, write a modified copy to a new location and re-point references. CoW can support snapshots and crash-safe commit protocols, but neither property is automatic: the engine still needs ordering, atomic root publication, synchronization, and reclamation rules.
Cursor
A stateful handle that yields query results one (or a batch) at a time instead of materializing the whole result set, bounding memory. Across the FFI it's an opaque handle valid only while its database is open, with a documented invalidation policy if the data changes mid-iteration.
Dirty page
A cached page whose in-memory copy differs from disk because it was modified. Must be flushed before it can be safely evicted or before commit.
Document
A self-describing, variable-length record — a serialized tree of fields and values, where each document carries its own structure (schemaless), unlike a relational row whose meaning comes from a fixed table schema. Stored as opaque bytes by the lower layers.
extern "C" / #[unsafe(no_mangle)] / #[repr(C)]
The three Rust attributes that produce C-compatible output: extern "C" selects the C calling convention, #[unsafe(no_mangle)] keeps the symbol name unmangled, #[repr(C)] lays out a struct the way C expects.
FFI (Foreign Function Interface)
The mechanism by which code in one language calls code in another, typically through a C ABI. Here: Dart calling your Rust engine.
ffigen
The Dart-side generator that reads a C header and emits Dart dart:ffi bindings — the symmetric counterpart to cbindgen (which generates the header from Rust). Like cbindgen it transcribes declarations and does not verify your ownership or safety contracts.
Freelist
The list of pages inside the file that are currently unused and available for reuse, so the file doesn't grow forever. SQLite's file format uses freelist trunk pages that contain references to freelist leaf pages. [src: SQLite freelist format]
fsync
An operating-system request to synchronize file data with its backing storage. It is necessary in many durability protocols, but the actual guarantee depends on the OS, filesystem, device, and whether related directory metadata also needs synchronization.
_id (primary key)
A document's required, unique primary key and the key of the collection's primary index. This curriculum simplifies it to an auto-assigned integer; MongoDB uses an ObjectId by default. The lookup is two-step: _id → record location → bytes.
Isolate (Dart)
Dart's unit of concurrency: a worker with its own memory and event loop that can't see another isolate's memory. Isolates communicate by passing messages over ports, not by sharing mutable state. Heavy database work is offloaded to a worker isolate to keep the UI responsive.
Isolation level
How much of concurrent transactions' effects a transaction may observe — a spectrum (Read Uncommitted → Read Committed → Repeatable Read → Serializable) defined by which anomalies it permits, not an on/off property. Serializable is the strongest, not the definition of "isolated."
Lock states (SQLite)
SQLite's rollback-journal locking ladder: UNLOCKED, SHARED (many readers), RESERVED (one intending writer, coexists with readers), PENDING (blocks new readers while draining), EXCLUSIVE (alone, for the actual write). Encodes "many readers OR one writer." [src: SQLite lockingv3]
Lost update
A concurrency anomaly where two transactions both read a value, modify it, and write back, so one update silently overwrites the other. The canonical reason a read-modify-write must be protected as a unit.
MessagePack
A compact, self-describing binary serialization format (type tags inline, like JSON but smaller). One candidate document codec, alongside JSON text and BSON. [src: MessagePack spec]
MVCC
Multi-version concurrency control: a write creates a new version rather than overwriting, so readers keep reading a stable older version while writers proceed — readers and writers don't block each other. Old versions are reclaimed once no snapshot needs them.
Opaque handle
A pointer the other language holds but cannot inspect — it only passes it back to your functions. In Rust: Box::into_raw hands out the pointer, Box::from_raw reclaims it. Keeps the C surface tiny and the internals private.
Overflow page
An extra page used to store the tail of a record too large to fit in its home page. The cell keeps a pointer to the overflow chain.
Ownership / move
Rust's rule that each value has exactly one owner; assigning or passing it moves ownership (the source can no longer be used) unless the type is Copy. When the owner goes out of scope, the value is dropped (freed).
Page
A fixed-size block (e.g. 4 KiB) that is the unit of I/O between the file and memory. The whole engine is organized around reading and writing whole pages.
Pager
The layer that turns a file into a numbered array of pages: it reads/writes pages, caches them, tracks dirty pages, and coordinates flushing and journaling. The foundation everything sits on.
panic
Rust's mechanism for failures that code does not return normally. Depending on the profile it unwinds or aborts. If an unwinding panic reaches extern "C", Rust aborts the process; extern "C-unwind" is the explicit unwind-permitting ABI. An embedded library may use catch_unwind to map unwinding panics to an error code, provided state remains valid.
Posting list
For a secondary index, the set of _ids sharing one field value, stored as the index entry's value. An alternative to encoding duplicates as composite (value, _id) keys.
Predicate / filter
A query expressed as a boolean test matches(doc) — a tree of comparisons combined with AND/OR/NOT. The evaluator applies it; a full scan plus the predicate is the correctness oracle every optimized plan must match.
Query planner
The component that chooses how to answer a query — most importantly, use a secondary index or full-scan — based on selectivity, then applies the rest of the predicate as a residual filter. This curriculum builds a rule-based planner, not a cost-based optimizer.
repr(C)
An attribute selecting Rust's C-compatible representation rules for a type. It is necessary when sharing concrete layout, but safety also requires matching field declarations, target ABI, valid values, alignment, lifetime, and ownership.
Result / Option / ?
Rust's error types: Result<T, E> is success-or-error, Option<T> is value-or-nothing, and ? propagates the error/None case to the caller. Rust has no exceptions; these are error handling.
Rollback journal
SQLite's original atomic-commit scheme: before modifying a page, copy its old contents to a journal file; on crash, restore from the journal. Contrast with WAL, which logs new page images. [src: SQLite atomic commit]
serde
Rust's standard serialization framework. #[derive(Serialize, Deserialize)] plus a format crate (e.g. serde_json) converts structs to/from bytes.
Schemaless
Structure lives in each document, not in an engine-enforced table schema: two documents in one collection may have different fields. The engine stores opaque bytes; meaning is the application's responsibility.
Secondary index
An index on a non-primary field, mapping a field's values to the _ids of documents holding them, so queries on that field avoid a full scan. Unique, composite (leftmost-prefix), and multi-entry (array) variants exist; each write must keep every index consistent.
Selectivity
The fraction of documents a condition keeps. Highly selective + indexed → use the index; low selectivity or no index → full scan. The core intuition a planner uses to pick an access path.
Send / Sync
Rust marker traits for thread safety: Send means a value can be moved to another thread; Sync means &T can be shared across threads. The compiler uses them to reject data races at compile time — memory safety, not logical database isolation.
Serializability
The strongest isolation level: any concurrent execution produces the same effect as running the transactions one at a time in some order. It is one level, not a synonym for "isolated."
Snapshot isolation
Each transaction reads as of a fixed point in time (a snapshot), so it sees a stable view while writers proceed. SQLite WAL mode gives readers a snapshot at the last commit before they began, so readers don't block the single writer and vice versa. [src: SQLite WAL]
Slotted page
A page layout for variable-length records: a slot/pointer array grows from one end, the record data grows from the other, free space sits in the middle. Lets records move during compaction without changing their slot index.
VFS (OS interface)
An abstraction over the operating system's file operations (open, read, write, sync, lock) so the engine doesn't bake in OS-specific calls. SQLite provides multiple VFS implementations. [src: SQLite architecture]
WAL (Write-Ahead Log)
A durability scheme: append every change to a log file and fsync that, before updating the main file. In SQLite WAL mode, a transaction commits when a frame carrying a commit marker is written; later checkpointing copies pages back to the database. [src: SQLite WAL]
